현재 대화 내용을 분석해서 도메인 지식을 아래 경로에 정리한다.

**도메인 문서 경로**: `/Users/jrlee/dev/repo/coder013/devlog/Domain/`

```
Domain/
├── Admin/
├── Inbound/
├── Inventory/
└── Outbound/
```

## 작업 순서

1. 도메인 분류 후 해당 디렉토리 확인
2. **기존 문서 확인** - 관련 파일이 있는지 Read로 확인
3. **문서 작성 원칙**
   - 파일이 없으면 신규 생성 (`kebab-case.md`)
   - 파일이 있으면 기존 내용에 추가/수정 (중복 작성 금지)
   - 구현 세부사항보다 **도메인 규칙과 개념** 중심으로 작성

4. **문서 구조 (권장)**

```markdown
# 도메인 개념명
> 관련 티켓, 작성일

## 핵심 개념
## 규칙 / 정책
## 예시
## 주의사항
## 관련 구현 위치
```

5. 작성 완료 후 어떤 파일에 무엇을 정리했는지 간략히 보고한다.
