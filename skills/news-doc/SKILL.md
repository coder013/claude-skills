# news-doc

웹 아티클을 한국어로 번역·정리하여 Obsidian에 저장한다.

## 저장 경로

`/Users/jrlee/dev/repo/coder013/devlog/Notes/Engineer News/`

## 실행 절차

### Step 1 - URL 콘텐츠 가져오기

`WebFetch`로 URL 내용을 가져온다.
가져온 원문의 **단어 수를 세어 예상 읽기 시간을 계산**한다.
- 영어 기준: 단어 수 ÷ 200 (분)
- 한국어 기준: 어절 수 ÷ 400 (분)

### Step 2 - 번역 및 정리

**번역 원칙**
- 자연스러운 한국어로 번역 (직역 금지, 의역 권장)
- 핵심 기술 용어는 `한국어 (원어)` 형식으로 병기 (첫 등장 시 1회만)
- 고유명사(제품명, 회사명, 사람 이름)는 원어 그대로 유지

**분량 기준**
- 원문 1,000 단어 이하: 원문에 충실하게 번역
- 원문 1,000~3,000 단어: 핵심 논지 중심으로 60~70% 압축
- 원문 3,000 단어 초과: 핵심만 남겨 50% 압축

**가독성 원칙**
- 원문의 섹션 구조 유지, 소제목이 없으면 흐름에 맞게 추가
- 중요한 주장·데이터·결론 문장은 **볼드** 처리
- 핵심 인용은 `>` 블록쿼트로 강조

### Step 3 - 태그 자동 분류

내용을 기반으로 아래 목록에서 1~3개 선택한다.

```
AI, Backend, Frontend, Cloud, Security, DevOps, Architecture,
Database, Mobile, Career, Product, Business, Research, Tool
```

주제가 명확하지 않으면 `General` 사용.

### Step 4 - 파일 작성 및 저장

**파일명**: 기사 제목을 한국어로 짧게 번역 (공백은 `-`로 대체)

**파일 형식**:

```markdown
---
tags: [태그1, 태그2]
source: {원문 URL}
date: YYYY-MM-DD
read_time: {N}분
---

## 핵심 요약

> {2~4줄로 이 글의 핵심 메시지를 압축}

---

## {섹션 제목}

{번역·정리된 본문}
```

`Write` 도구로 파일을 저장한다.

### Step 5 - 결과 보고

```
저장 완료: {파일명}
태그: {태그 목록}
예상 읽기 시간: {N}분
```
